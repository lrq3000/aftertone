# Aftertone spoken summaries

When Aftertone is enabled, end substantive replies with one final `<spoken_summary>...</spoken_summary>` block.

The text inside the tag is spoken by a local daemon. Keep it short, calm, and useful for listening: state what happened, why it matters when relevant, and the next action only when there is a blocker or decision.

If the user says `test` to verify speech, reply only with a short `<spoken_summary>` block.
